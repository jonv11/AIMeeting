## âš™ï¸ How to Maintain This Structure

### When Adding New Documentation
1. **Choose the right folder** based on content type
2. **For status reports**: Create `/docs/reports/YYYY-MM-DD/` folder with date
3. **For planning**: Use `/docs/planning/v0.X/` folder with version
4. **Update navigation**: Edit relevant hub (docs/README.md, etc.)
5. **No duplication**: Link instead of copying content

### When Archiving Documentation
1. **Move to `/docs/archive/`**
2. **Update archive/README.md** with reason & replacement
3. **Update navigation hubs** to link to replacement
4. **Verify no broken links**

### When Updating Docs
- **Update source file only** (not copies)
- **Update all links** if moving or renaming
- **Keep timestamps** for reports
